Dev notes
Board picker (the critical surface)
  • Required board dropdown — lists the user's boards; token's boardId set from selection (server-stored, the security boundary)
  • Connection name auto-fills from board name + " — Claude Code"; editable but pre-filled
  • Reassurance framing throughout: "only this board / nothing else" — the green shield box restates it before generating
  • "One connection = one board" stated as a safety feature, not a limit
Post-generation
  • Token shown once (raw token never stored — same as Phase A), copy button, pre-built claude mcp add command
  • Plain-language access summary with the 🤖 mark: what Claude can do, scoped to this board
Hard gate
  • Shown when no Claude subscription/API key — hard gate, no trial/fallback in v1
  • Two tones provided — pick A (warm) or B (minimal)
Tokens / guardrails
  • Canonical tokens, glass on the picker dropdown. Board-scope enforced server-side (a token must not be able to act on any board other than its boardId)
