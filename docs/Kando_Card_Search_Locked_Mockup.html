Dev build notes — Option B + 4 folded fixes
One card search, two scopes. Reuses filterCards (board-agnostic, confirmed), the open-to-card primitive, and the Alerts mobile-takeover. Four latent issues fixed in the same pass.
Core — scope toggle + two modes
  • Add a scope toggle to the card search: This board (default) / All boards. Default = This board (matches today, and it's the in-place-filter mode users know)
  • This board = existing in-place filter, unchanged: filterCards(currentColumn.cards, query, filters), dim non-matches in columns, highlight matches
  • All boards = results list, NOT in-place (off-screen boards can't be filtered in place). Flatten across boards: boards.flatMap(b => b.columns.flatMap(c => c.cards)) (memoize with useMemo keyed on boards+query), run the same filterCards, then group results by board with board-name headers. Each result shows column + priority left-border + match highlight
  • Tapping an All-boards result → open-to-card primitive (switchBoard(boardId) + setActiveCardId(cardId)) — the confirmed navigation path
Four folded fixes (all in search files)
  • Stale query on board switch: searchQuery persists across switchBoard today, silently filtering the new board. Clear searchQuery/filters on board switch (or scope to the board) so switching boards doesn't carry a stale filter
  • Ctrl+/ not wired: the tooltip advertises it but no listener exists. Wire Ctrl+/ in useKeyboardShortcuts.ts to focus/open the search bar
  • Collapsed badge misses text-only filters: hasFilters checks only priority/overdue, so a text-only active search shows no badge when collapsed. Include a non-empty searchQuery in the badge condition
  • Scope indicator: the toggle itself is the indicator — the active scope is always visible, fixing "user doesn't know they're searching this-board-only"
Mobile + modernization
  • Restyle the desktop search to glass/tokens (it's legacy dark:-paired today). Wire the dead bottom-tab "Search" button to a full-screen mobile takeover (Alerts pattern: back arrow, tab bar hidden, scope toggle + grouped results)
  • Canonical tokens only. Reuse the Alerts takeover structure. Respect prefers-reduced-motion
Explicitly OUT of scope
  • Board search (switcher) — modernize styling only, behavior unchanged (separate tool)
  • The 3 built-but-hidden filters (tags/due-date/assignees) — leave hidden, track for a future filters pass
  • Calendar/Analytics staying unaffected by search — confirmed intentional, leave as-is
