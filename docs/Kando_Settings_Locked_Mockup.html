Dev build notes — new surface
Settings is a new full-page surface reached from the avatar menu. No logic changes to existing features — but the three items moving here from the overflow are rebuilt in place, not patched.
Structure
  • Full-page view: left .set-nav (218px) + right .set-content pane. Entered from the avatar (R ▾) menu — add Settings there alongside Profile / Log out
  • Seven sections: Account · Appearance · AI & Integrations · Notifications · Subscription · Advanced · About & Legal
  • Each section = elevated .set-card on --s2 with Level-2 shadow; .card-label = 11px/600 uppercase --purple-light; fields/rows stay subordinate (field-label treatment, not section headings)
  • Mobile: .set-nav wraps to a top list, content stacks single-column, 44px touch targets (see @media(max-width:720px))
AI & Integrations — the token screen
  • Restyle only — keep the existing token-generation logic and the exact command string your code already produces. Do not change what the command outputs; just wrap it in the .cli block treatment (mono, inset dark bg, copy button, horizontal scroll)
  • Generate Token = primary glow button; generated command shown in .cli with a one-tap copy; "shown once / revoke below" caveat line
  • Active Tokens list with per-token Revoke (danger-outline button) — UI affordance for the MCP revocation work already on the roadmap
Moved from the overflow — REBUILD, don't patch
  • System Health regressed in the toolbar refactor (renders unthemed, content out of view). Rebuild it fresh inside Advanced → it should open themed and functional, matching its pre-regression behavior
  • Board Recovery and Advanced Recovery move into Advanced as .row items with an Open action
Relocated here
  • Legal links (Terms / Privacy / Cookie / AI-Usage) removed from the mobile board view live in About & Legal as .link-row items
  • Subscription is an honest placeholder while Stripe is test-mode only — show current plan; the Manage button is visibly not-yet-active, not a fake working flow
Tokens / theming
  • Theme tokens only, no hardcoded colors. Token-styled inputs (rgba(22,20,18,.6) bg, --border2, purple focus ring). Glow only on the primary action
