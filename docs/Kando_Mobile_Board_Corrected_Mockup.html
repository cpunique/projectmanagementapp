Same layout. In standalone PWA there's no browser bar — the tab bar IS the bottom.

1 · Centered, no left bias

The column scrollbar is hidden (scrollbar-width:none + webkit), so it can't consume the right edge and push cards left. Cards sit with equal gutters on both sides.

2 · Bigger cards + gutters

Gutter increased to 20px, card padding to 18px, title 15px, more internal spacing — cards breathe more on all sides, matching the zoomed reference.

3 · No footer artifacts

Terms / Privacy / Cookie / consent links are removed from the board view. They belong in a Settings or account screen, not under the mobile board. The board ends with the cards + Add Task.

4 · Tab bar anchored to bottom

Board / Calendar / Search / Alerts is pinned to the true bottom of the screen (flex layout: scrollable board fills the middle, tab bar fixed at the bottom). In standalone PWA there's no browser chrome below it. The "+" add button is a rounded square (14px radius), matching the desktop add-column button.

Top edge

App sits below the OS status bar (correct). In standalone PWA the top bar background extends up behind the status bar via env(safe-area-inset-top) — no awkward gap.

No duplicate project name

The project name was appearing twice (top-bar selector + page title). On mobile, removed it from the top bar — the top bar is just logo + actions. The page title below carries the name and the ▾ dropdown for switching projects.

Bottom edge

Tab bar respects env(safe-area-inset-bottom) for the home indicator. The DuckDuckGo bar you see while testing in-browser disappears once installed as a PWA.
