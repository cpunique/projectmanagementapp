Dev build notes — new surface
First-run welcome modal. First surface built entirely in canonical token names — no aliases.
Trigger & lifecycle
  • Fires once for brand-new users — gate on a per-user flag (e.g. hasSeenOnboarding on the user record). Set the flag when dismissed by any path so it never reappears
  • Both the primary button and "I'll explore on my own" and the close × all dismiss + set the flag — never trap the user inside onboarding
Actions
  • Create your first board = primary glow button → dismiss modal + trigger the existing create-board flow (do not rebuild it). This is a free action — board creation isn't gated
  • I'll explore on my own = subtle ghost button → just dismiss
The AI tease — show, don't gate
  • Tease board generation only. The PRO pill is honest signalling, not a gate — clicking the primary button must NOT hit a paywall. Users meet the Pro gate naturally later at "Generate Tasks"
  • Do NOT mention prompt-generation or the MCP key here — out of scope for first-run
Glass-modal pattern (the recurring trap)
  • backdrop-filter dies under transform. Animate an outer wrapper (transform/opacity); put the glass on an inner static .modal with no transform — same split as the other modals
  • Panel: rgba(42,37,34,.7) + blur(24px) saturate(1.2), border rgba(255,255,255,.09), radius 18px, Level-3 shadow; full-screen backdrop rgba(0,0,0,.5) behind
Tokens & responsive
  • Canonical token names only — --surface-2, --surface-3, --border-2, --purple-l, --green/--amber/--red. No --s2 / --purple-light aliases anywhere
  • Mobile: panel becomes a near-full-width bottom sheet (radius top-only), 44px touch targets, respect safe areas. Respect prefers-reduced-motion; solid-surface fallback where backdrop-filter unsupported
