How sharing works
  • Editors can view and modify all cards and columns
  • Viewers can only view the board (read-only)
  • The green dot shows who's currently viewing the board
  • Changes sync across everyone every 10–30 seconds

Two entry points (kept — collaboration intent)

This same modal opens from BOTH the board toolbar Share button and the card-modal Share Board button — the deliberate multi-surface access to promote collaboration. One shared modal component, surfaced from two places.

Dev build notes

Backend — reuse, don't rebuild

The viewer/editor + collaborators[] / sharedWithUserIds model and share API routes ALREADY exist on current main and are server-enforced (Firestore rules). This is UI-only — wire the modernized modal to the existing invite / role-change / revoke endpoints. Do NOT reimplement sharing logic or touch the enforcement.

UI

Glass modal (canonical tokens, Level-3 shadow, frozen header). Invite row: email + role dropdown (Viewer default, Editor option) reusing the glass Dropdown + Invite. Collaborator rows: avatar w/ presence dot, name + online state, role pill (Owner=purple, Editor=blue, Viewer=neutral), Remove (revoke) on collaborators — NOT on owner. Empty state when no collaborators. Modernized "How sharing works" tips panel. Same modal opens from both entry points. No dark:/hardcoded colors.
